# EtherDex Roadmap

## Positioning
EtherDex is a COLLECTION TRACKER, not just a price checker.
"The Collector's Index" — your collection, indexed, valued, tracked over time.
Differentiation = the collection-ownership experience, NOT data depth.

## Status — Done
- Search, filters, autocomplete, self-healing cache
- Accounts + per-user watchlist (Clerk auth)
- Spreadsheet export
- Card detail modal
- EtherDex branding (logo, Orbitron, "The Collector's Index")
- Card condition selector (6 conditions + descriptions)
- Phase 8: price history recording + chart

## Next Up — Improvements (recommended order)

### Cheap wins first
1. Test the app on mobile — fix any responsive issues (filter bar especially)
2. Polish onboarding / empty states — guide first-time users
3. Add "quantity owned" to the watchlist (turns it into a real collection)

### Big bets (in order)
4. Collection dashboard — total value, gainers/losers, value-over-time chart
5. Price alerts — "your card hit $X" / weekly movement digest (needs notifications)
6. Set completion tracking — "47 of 102 cards in Plasma Blast"

## Larger Phases (later)
- Phase 7: Payments + 4 tiers (Stripe). Start with planning + Stripe setup step.
- Phase 9: Card scanning. Bundle the condition price-multiplier feature here.

## Notes / Do NOT do
- Don't try to out-data the giants — compete on experience + collection angle.
- Don't build a marketplace (buying/selling) — scope/fraud trap.
- Evaluate a paid price-history source LATER — post-revenue, check licensing first.
- Verify condition definitions vs TCGplayer's official guide before launch.
